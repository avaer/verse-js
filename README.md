# verse-js

A self-contained browser IDE that edits, **runs, and debugs** a subset of
[Verse](https://dev.epicgames.com/documentation/en-us/uefn/verse-language-reference),
Epic Games' programming language for UEFN/Fortnite — no backend, no UEFN
install, everything happens in the browser tab.

![Verse IDE](https://img.shields.io/badge/verse-ide-3fa7ff) ![Next.js 16](https://img.shields.io/badge/next.js-16-black) ![React 19](https://img.shields.io/badge/react-19-61dafb)

## Features

- **Monaco editor** with the official Verse TextMate grammar (full syntax
  highlighting via `vscode-textmate` + `vscode-oniguruma`), diagnostics as
  squiggles, and a dark Verse theme
- **Run button** (F5): preprocess → parse (Peggy) → semantic analysis →
  tree-walking async interpreter, with output streaming into the console panel
- **Debugger**: click the gutter to set breakpoints, then Debug to pause with
  a variables panel and call stack; Step Over / Step Into / Step Out /
  Continue; Stop cancels any run, including mid-`Sleep` and infinite `loop:`
- **Verse failure semantics**: `if (...)` conditions and the left operand of
  `or` are failure contexts; writes made inside them are journaled and rolled
  back when the context fails (modelled on VerseVM's speculative execution)
- **`suspends` natives**: `Sleep` maps onto awaited timers, so coroutine-style
  countdowns stream to the console in real time
- Multi-file workspace (create/rename/delete `.verse` files) persisted to
  localStorage, resizable panes, streaming console with timestamps and
  click-to-jump error locations

## Getting started

```bash
pnpm install
pnpm dev        # http://localhost:3000
pnpm test       # vitest suite (parser/interpreter/semantics/rollback/debugger)
pnpm build      # production build
```

The editor seeds four example files:

| File | Demonstrates |
| --- | --- |
| `hello-world.verse` | `creative_device` + `OnBegin` entry point, `Print`, string interpolation |
| `failure-rollback.verse` | `<decides>` functions, failure contexts, transactional rollback |
| `sleep-countdown.verse` | `Sleep` + `loop:`/`break`, real-time output, Stop mid-run |
| `random-dice.verse` | `/Verse.org/Random` natives, `for` ranges — good breakpoint demo |

## Supported language subset

Declarations (`x := ...`, `var x : int = ...`), `set` with compound operators,
`int`/`float`/`string`/`logic`/arrays, string interpolation, `if`/`else`
(indented, inline, `then`, and curly-brace forms), `loop`/`break`, `for` over
ranges, functions with `<decides>`, classes extending `creative_device` (the
`OnBegin` method is the program entry point), failure contexts with rollback,
and the native modules `/Verse.org/Verse` (Floor, Ceil, Round, Int, Mod,
Quotient), `/Verse.org/Random`, and `/Verse.org/Simulation` (Sleep).

Not implemented: UEFN device/player APIs, custom (non-device) classes and
structs, maps, `race`/`sync`/`spawn` concurrency, generics, `option` types.

## Architecture

```
src/verse/           language core (framework-free, runs in Node + browser)
  grammar/           Peggy grammar (statements carry source locations)
  parser.js          generated by `pnpm build:parser`
  preprocess.js      indentation -> `end` injection, with a line map back
                     to original source for breakpoints/errors
  semanticAnalysis.js compile-time checks (unknown identifiers, 3512
                     failure-context enforcement, ...) with source locations
  interpreter.js     async tree-walking interpreter: streaming output,
                     per-statement debug hook, transactional writes
  runtime/failure.js VerseFailure + Transaction (rollback journal)
  runtime/libraries.js native module registry (Sleep, Random, math...)
  debug/DebugController.js breakpoints, stepping, pause/resume, cancellation
src/monaco/          Monaco loader config + Verse TextMate registration
src/ide/             React components: Ide shell, EditorPane, Console,
                     DebugPanel, FileSidebar, EditorTabs, Toolbar
app/                 Next.js app router entry (IDE is client-only)
```

The interpreter awaits a controller hook before every statement, which is how
breakpoints, stepping, Stop, and event-loop yielding (so hot loops can't
freeze the tab) are all implemented without workers.
